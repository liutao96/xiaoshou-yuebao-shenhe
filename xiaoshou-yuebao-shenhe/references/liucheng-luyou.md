# 初审刀、复审刀、深度刀路由

## 三把刀定位

### 初审刀

用于部门第一次提交月报，或用户只发一条飞书月报链接。

目标：

- 找出数据不匹配、口径不统一、逻辑不清晰、前后矛盾；
- 生成初审评分；
- 生成老板追问 5 问；
- 判断 A/B/C/D；
- 需要时输出打回整改单和主管整改版。

### 复审刀

用于部门整改后再次提交。

必须有材料包：

- 整改后月报；
- 上次 AI 初审结果或主管整改版；
- 整改单逐条回复；
- 老板追问 5 问答复；
- 如涉及口径、费用、品项、动作问题，对应补充表也要提供。

目标：

- 检查上次问题是否逐条整改；
- 检查老板追问是否回答到位；
- 判断是否可以提交老板；
- 通过后生成老板一页纸正式版。

### 深度刀

用于重点部门、问题部门、老板会前、季度复盘，或用户明确要求“深度经营诊断”“14 个模块”“不要简单回答”。

目标：

- 按 14 模块完整展开；
- 输出数据审计、经营诊断、管理风险、下月策略；
- 不只判断月报质量，也帮助老板看经营本质。

## 默认路由

| 用户输入 | 默认处理 |
|---|---|
| 只发飞书月报链接 | 初审刀 |
| 说“审核这个月报” | 初审刀 |
| 说“复审/整改后/重新提交” | 复审刀 |
| 说“深度审核/经营诊断/老板会/14 模块/不要简单” | 深度刀 |
| 只查某个口径、费用率或数据冲突 | 专项检查，不输出完整分流 |

## 不允许的简化

- 不能只因为发现一个明显问题，就跳过其他核心模块。
- 不能只输出“C 类、70 分、补充数据”。
- 不能把 C 类主管整改版写成普通建议。
- 不能把老板一页纸当成全部审核报告。
- 不能把业务表现评分和月报质量评分混在一起。
