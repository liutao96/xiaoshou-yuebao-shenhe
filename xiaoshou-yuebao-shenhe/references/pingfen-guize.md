# 销售月报业务口径与评分规则_V1.1_口径过渡版

## 一、最终统一方向
公司未来可以逐步统一到净成交额，但当前版本先保证销售月报能稳定运行。

净成交额 = 支付成交额 / GSV - 退款金额。

当前审核采用：现有主口径清楚且前后一致即可。净成交额是可选增强项；口径桥接只在多个核心金额冲突并影响经营判断时使用。

## 二、各小组过渡期主口径表
| 小组类型 | 当前可用主口径 | 未来参考口径 | 有则分析的辅助口径 |
|---|---|---|---|
| 平台店铺组：天猫、京东、拼多多、电商一组、电商二组 | 销售回款 + 平台销售额/GSV辅助 | 净成交额 | 开票、发货；费用归属销售额、退款只在原报告提供或本月异常相关时分析 |
| 抖音商城组 | GSV / 支付成交额 | 净成交额 | 店铺自营、达人/KOL/KOC拆分、商品卡、直播、短视频、图文 |
| KOC/KOL/达人/短视频组 | GSV | 净GSV / 净成交额 | 佣金、投流、坑位费、单达人ROI、退款 |
| 新零售/分销/商超/经销组 | 销售回款 | 回款为管理主口径，净成交额做公司经营辅助 | 开票、发货、客户动销、账期；客户库存只在原报告提供或库存/压货直接相关时分析 |
| 财务视角 | 销售回款 / 开票 / 结算 | 不替代经营主口径 | 现金流、开票、结算 |
| 仓储履约视角 | 发货金额 / 出库金额 | 不替代经营主口径 | 履约、出库、备货 |

## 三、核心口径定义
### 1. 当前主口径销售额
过渡期各小组用于汇报目标完成的主口径。必须标注类型：净成交额 / 销售回款 / GSV / 支付成交额 / 开票 / 发货 / 其他。

### 2. 支付成交额 / GSV
平台或内容成交口径，适合平台店铺、抖音、KOC、KOL、达人、短视频组。不能直接代表财务回款。

### 3. 退款金额与退款率
退款金额和退款率不是常规必填项。原报告已提供时核验公式和趋势；原报告未提供时，不要求业务另做表，也不要求承诺补齐时间。若本月明确发生重大退款/售后问题并影响销售结论，只补一段总量和原因说明即可。

### 4. 净成交额
可作为未来经营口径或报告加分项，当前不作为通过门槛。

### 5. 销售回款
财务回款口径，适合平台店铺、商超、分销、新零售。不能直接替代平台成交或投放效果。

### 6. 开票金额
财务开票口径。适合新零售、分销、商超、经销客户。不能直接等同销售业绩。

### 7. 发货金额 / 出库金额
履约和仓储口径。适合看供货节奏、客户备货、品项出库结构，不能直接当作销售结果。

### 8. 费用归属销售额
用于计算推广 ROI 或费用率的销售额，可能不等于部门总销售额。必须说明分母范围。

## 四、费用率规则
当前审核：
费用不是常规必填项。只有原报告出现费用率、ROI、预算费用、费用表或明确费用结论时，才审核费用金额和所用销售分母。没有净成交额不影响使用现有口径，可标注分母为：
- 当前主口径销售额；
- 平台销售额/GSV；
- 推广归属销售额；
- 销售回款；
- 其他。

以下指标按报告已有内容选用，不要求同时展示：
| 指标 | 公式 | 用途 |
|---|---|---|
| 经营费用率 | 营销费用 / 净成交额 | 老板看真实经营效率 |
| 过渡费用率 | 营销费用 / 当前主口径销售额 | 过渡期沿用 |
| 推广ROI | 推广归属销售额 / 现金投放 | 投放效果 |
| 总费比 | 佣金+投流+坑位+折扣 / GSV或净成交额 | 内容达人效率 |

## 五、退款和净成交额评分规则
| 情况 | 评分处理 |
|---|---|
| 未提供退款金额/退款率/净成交额 | 不扣分、不封顶；写明“报告未提供，本次不作为基础审核必填项” |
| 只说明当前销售口径，无法确认是否含退款 | 正常评分；必要时建议补一句“已含/未含/暂不确定” |
| 已提供退款金额和退款率 | 正常评分 |
| 已提供净成交额且可复算 | 可作为数据完整性加分依据 |
| 本月存在重大退款问题却用含退销售额得出明显错误结论 | 按结论失真程度扣分，必要时要求补充总量说明 |

## 六、毛利和库存规则
毛利率、毛利金额、库存金额、库存状态和品项退款率均为可选字段，不是销售月报基础审核门槛。

如果月报仅凭销售额提出“无条件加码”，审核意见可写“主管内部确认收益与供货条件后再加码”，但不得因此要求运营补表、补毛利档位、补库存状态或直接打回。

## 七、品项判断规则
品项审核重点看销售贡献、异常增长/下滑原因和下月动作。不得默认要求补“品相分类”。只有原报告已经使用战略品、利润品、冲量品、引流品、清仓品、新品等分类，或主管明确要做品项策略复盘时，才沿用该分类。

## 八、评分规则
总分 100 分：
| 维度 | 分值 |
|---|---:|
| 问题归因质量 | 30 |
| 数据真实性与一致性 | 25 |
| 执行质量与闭环 | 20 |
| 经营结果表现 | 15 |
| 下月计划可落地性 | 10 |

## 九、最高分限制
| 情况 | 最高分 |
|---|---:|
| 总销售额和分渠道合计差异超过 5% 且无解释 | 60 |
| 店铺销售和达人/KOL/KOC销售混用且无说明 | 65 |
| 原报告已有费用率/ROI/预算费用或明确费用结论，但费用率分母不清且影响核心费用结论 | 80 |
| 原报告多处费用率分母冲突，导致费用结论不可信 | 70 |
| 关键数据前后冲突 | 70 |
| 没有上月动作闭环 | 80 |
| 只有结果没有归因 | 65 |
| 有明显模板错误、月份错误、字段未改 | 80 |
| 疑似重大数据造假或故意隐瞒 | 50，并建议重做 |

## 十、过渡期分流判断
1. 当前主口径明确、关键数据前后一致，即使没有退款、净成交额、毛利或库存，也可正常判 A 或 B。
2. 未标注当前主口径但从表格可明确判断，优先要求补一句口径备注，通常判 B；多个核心金额冲突并影响结论且无法解释时才判 C。
3. 主销售额无法判断 + 核心表格大量冲突：判 D。
4. 不得仅因可选字段缺失触发最高分限制或 C/D 分流；费用、退款、毛利、库存、复购、品相分类未提供时均按可选/条件项处理。
