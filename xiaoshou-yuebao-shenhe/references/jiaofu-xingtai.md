# 最终交付形态规则

## 总原则

月报审核结果不能只停留在几句聊天文字。正式初审、深度诊断、C/D 打回整改，默认要形成可留档的审核报告。

## 默认交付

### 用户只发飞书链接

默认输出三层：

1. 聊天区部长首屏回复：给出读取状态、内部审计分流结果、评分、是否可提交老板、最大问题、14 模块速览、部长下一步动作。
2. 完整 14 模块初审报告：如果当前环境可写文件，生成 Markdown 审核报告；如果不可写文件，则在聊天区输出完整 Markdown 正文。
3. 最后一段可转发整改指令：给主管或部门直接照着改，不能只写“见报告”。

部长或管理者只贴飞书链接时，默认就是“部长要看初审结果”。不能只输出简单摘要，必须先在聊天区给 14 模块速览，再给完整 14 模块文档路径。

### 完整 14 模块审核

必须生成完整 Markdown 报告。聊天区不用把全部长文重复一遍，但必须按 `buchang-shouping-huifu.md` 给每个模块至少一句速览，不能只放关键结论和报告路径。

完整报告前部必须加入“本报告生成逻辑”，说明读取范围、审核路径、14 模块框架、评分权重、最高分限制和 A/B/C/D 分流规则。

### C 类或 D 类

必须生成主管整改包。聊天区展示分流、分数、5 个必须整改问题和提交要求；完整报告保留口径桥接、费用整改、品项整改、动作闭环、老板追问、复审标准。

### A 类或 B 类

默认聊天区输出老板一页纸或轻量补充清单。若用户说“发给老板/留档/生成文档”，生成 Markdown 或 Word 文档。

### 复审

聊天区输出复审结论、是否建议提交老板、最多 5 条剩余整改项。复审通过并需要留档时，生成老板一页纸正式版。

## Markdown 与 Word 的选择

| 场景 | 首选形态 | 说明 |
|---|---|---|
| Codex/AI 内部审核 | Markdown | 便于复制、版本管理和继续复审 |
| 部长贴链接快速看结果 | 聊天区 14 模块速览 + 14 模块 Markdown 报告 + 可转发整改指令 | 不打开文档也能知道各模块审出了什么 |
| 给主管整改 | Markdown 报告 | 表格清楚，便于逐条回复 |
| 给老板月会 | Word 文档或 Markdown 转 Word | 更适合正式流转 |
| 飞书文档沉淀 | Markdown 正文或飞书 Docx | 如果有写入权限，可创建飞书文档 |

## 文件命名

生成本地报告时使用：

```text
outputs/feishu_monthly_audit/YYYY-MM_部门或小组_销售月报AI初审报告.md
outputs/feishu_monthly_audit/YYYY-MM_部门或小组_销售月报主管整改包.md
outputs/feishu_monthly_audit/YYYY-MM_部门或小组_销售月报深度经营诊断.md
outputs/feishu_monthly_audit/YYYY-MM_部门或小组_销售月报复审结果.md
```

如果无法识别部门或月份，用文档标题和当前日期代替。

## Word 文档生成规则

只有在以下情况生成 Word：

- 用户明确说要 Word、docx、正式发老板、正式发主管；
- 当前环境有可用文档处理能力；
- 报告已经先形成 Markdown 底稿。

Word 生成失败时，不中断审核结论，交付 Markdown 作为正式底稿，并说明未能生成 Word 的原因。

## 不允许

- 不允许只输出一句“建议打回整改”。
- 不允许把完整审核只放在心里，最终不给用户可追溯材料。
- 不允许为了正式感生成 Word，但内容仍然是简化结论。
- 不允许给部长的初审报告缺少“本报告生成逻辑”。
- 不允许部长首屏回复只写 3 条硬伤和一个报告路径。
- 不允许用“详见报告”替代 14 模块速览。

## 最后一段可转发整改指令

每次正式初审结束后，聊天区最后必须给一段可直接复制转发给主管/部门的文字。

要求：

- 不写成 AI 自我说明；
- 不写复杂提示词；
- 语气像部长布置整改；
- 明确是否需要重写全文，还是补充关键说明；
- 明确补哪些表、哪些口径、哪些动作；
- 明确补完后提交复审。

模板：

```text
本次 AI 初审结论：建议先补充整改后再提交老板，不需要重写全文。请重点补充：1）销售主表、费用表、订单导出之间的口径桥接；2）费用率超预算的复算说明和整改动作；3）退款金额、退款率及对 GSV 的影响；4）下月目标调整原因和达成路径；5）重点品项、直播间和商品卡的具体改进动作。补充完成后，请连同本次 AI 初审报告一起提交复审。
```

如果报告为 A/B 类，可改为：

```text
本次 AI 初审结论：整体可以进入老板审阅，但提交前请补齐口径说明和关键差异解释，避免老板追问时无法当场回答。请按初审报告中的待补清单补充后，再提交最终版。
```
