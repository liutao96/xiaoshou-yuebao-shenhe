# 复审简化入口规则

## 目的

部长或主管整改后使用复审时，不能要求他们一开始就准备复杂材料包。默认让他们“发整改后月报链接 + 一句话”即可。

复杂材料包是内部完整标准，不是部长端首要话术。

## 推荐入口

部长或主管可以直接发送：

```text
这是修改后的月报，请做复核：<飞书链接>
```

或：

```text
这个月报已按上次意见修改，请复审：<飞书链接>
```

或：

```text
整改后的月报：<飞书链接>
```

这些话都必须识别为复审刀，不要当成首次初审。

## 内部处理顺序

收到“修改后、整改后、复审、复核、重新提交、已按意见修改”等关键词时，按以下顺序处理：

1. 先读取整改后月报链接。
2. 自动寻找上次初审内容。
3. 如果找到了上次初审内容，直接做复审，不要求用户重复提供。
4. 如果找不到上次初审内容，再提示用户补充“上次 AI 初审报告或主管整改版”。
5. 如果整改后月报本身没有逐条回复，但能通过前后文档对比判断，也先输出“可复核项”和“仍缺逐条说明项”，不要直接中止。

## 上次初审内容的自动检索范围

优先按以下顺序找：

1. 当前会话上下文里的上次 AI 初审结论、主管整改版、老板追问 5 问。
2. 当前工作区输出目录，例如：
   - `outputs/feishu_monthly_audit/`
   - `outputs/`
   - 当前项目内包含“初审报告”“主管整改包”“销售月报AI初审报告”的 Markdown / Word 文件。
3. 用户刚发的飞书文档中是否附带或引用了上次整改意见。
4. 同一部门、同一月份、同一文档标题附近的历史报告。

如果找到多个候选，不要自行混用。列出最可能的 2-3 个候选，让用户确认使用哪一个。

## 找不到上次初审时的回复

不要长篇解释，也不要要求全套材料包。只问最关键的一件事：

```text
我已读取到整改后月报，但当前会话/本地输出目录里没有找到对应的上次 AI 初审报告。为了做正式复审，请把上次 AI 初审报告或主管整改版发我；如果只是快速复核，我也可以先按当前月报做“整改后试审”。
```

## 复审输出

复审输出不要重新完整铺开 14 模块，除非用户要求。

默认输出：

1. 是否找到上次初审依据；
2. 是否读取到整改后月报；
3. 上次主要问题逐条整改结果；
4. 仍未解决的问题，最多 5 条；
5. 是否可以提交老板；
6. 如果通过，生成老板一页纸正式版；
7. 如果未通过，给剩余整改指令。

## 判断边界

- 找到上次初审依据 + 整改后月报：可以做正式复审。
- 只找到整改后月报，找不到上次初审依据：只能做整改后试审，不能声称正式复审通过。
- 如果整改后月报没有逐条回复，但修改内容能覆盖上次问题，可以判“部分通过/需补说明”，不要机械打回。
- 如果核心口径、费用、退款、目标解释仍没补齐，应继续打回。
