# 销售月报业务口径与评分规则_V1.1_口径过渡版

## 一、最终统一方向
公司最终经营主口径建议统一为：净成交额。

净成交额 = 支付成交额 / GSV - 退款金额。

但当前处于口径统一过渡期，不建议一刀切要求所有小组立即使用净成交额。过渡期应采用：当前主口径 + 口径桥接 + 逐步补齐净成交额。

## 二、各小组过渡期主口径表
| 小组类型 | 过渡期主口径 | 未来主口径 | 必须保留辅助口径 |
|---|---|---|---|
| 平台店铺组：天猫、京东、拼多多、电商一组、电商二组 | 销售回款 + 平台销售额/GSV辅助 | 净成交额 | 开票、发货、费用归属销售额、退款 |
| 抖音商城组 | GSV / 支付成交额 | 净成交额 | 店铺自营、达人/KOL/KOC拆分、商品卡、直播、短视频、图文 |
| KOC/KOL/达人/短视频组 | GSV | 净GSV / 净成交额 | 佣金、投流、坑位费、单达人ROI、退款 |
| 新零售/分销/商超/经销组 | 销售回款 | 回款为管理主口径，净成交额做公司经营辅助 | 开票、发货、客户动销、客户库存、账期 |
| 财务视角 | 销售回款 / 开票 / 结算 | 不替代经营主口径 | 现金流、开票、结算 |
| 仓储履约视角 | 发货金额 / 出库金额 | 不替代经营主口径 | 履约、出库、备货 |

## 三、核心口径定义
### 1. 当前主口径销售额
过渡期各小组用于汇报目标完成的主口径。必须标注类型：净成交额 / 销售回款 / GSV / 支付成交额 / 开票 / 发货 / 其他。

### 2. 支付成交额 / GSV
平台或内容成交口径，适合平台店铺、抖音、KOC、KOL、达人、短视频组。不能直接代表财务回款。

### 3. 退款金额与退款率
退款金额用于计算净成交额。退款率 = 退款金额 / 支付成交额或GSV。当前阶段如果暂未提供，必须写明数据来源缺失和预计补齐时间。

### 4. 净成交额
最终经营主口径，适合老板一页纸长期使用。过渡期可暂不作为唯一主口径，但应逐步补齐。

### 5. 销售回款
财务回款口径，适合平台店铺、商超、分销、新零售。不能直接替代平台成交或投放效果。

### 6. 开票金额
财务开票口径。适合新零售、分销、商超、经销客户。不能直接等同销售业绩。

### 7. 发货金额 / 出库金额
履约和仓储口径。适合看供货节奏、客户备货、品项出库结构，不能直接当作销售结果。

### 8. 费用归属销售额
用于计算推广 ROI 或费用率的销售额，可能不等于部门总销售额。必须说明分母范围。

## 四、费用率规则
最终统一：
营销费用率 = 营销费用 / 净成交额。

过渡期：
如果没有净成交额，必须标注费用率分母：
- 当前主口径销售额；
- 平台销售额/GSV；
- 推广归属销售额；
- 销售回款；
- 其他。

建议同时展示：
| 指标 | 公式 | 用途 |
|---|---|---|
| 经营费用率 | 营销费用 / 净成交额 | 老板看真实经营效率 |
| 过渡费用率 | 营销费用 / 当前主口径销售额 | 过渡期沿用 |
| 推广ROI | 推广归属销售额 / 现金投放 | 投放效果 |
| 总费比 | 佣金+投流+坑位+折扣 / GSV或净成交额 | 内容达人效率 |

## 五、退款数据评分规则
| 情况 | 评分处理 |
|---|---|
| 未提供退款金额/退款率，也没有说明 | 最高 75 分 |
| 暂未提供退款，但说明数据来源和补齐时间 | 最高 85 分 |
| 已提供退款金额和退款率 | 正常评分 |
| 已提供净成交额 | 正常评分并可加分 |

## 六、毛利和库存规则
当前阶段不要求完整毛利率和库存金额，但重点品项至少补充：
- 毛利档位：高 / 中 / 低 / 未提供。
- 库存状态：充足 / 偏紧 / 缺货 / 积压 / 未提供。

缺毛利和库存时，不能直接判断某品项一定值得加码。

## 七、品相分类
统一使用 6 类：
1. 战略品；
2. 利润品；
3. 冲量品；
4. 引流品；
5. 清仓品；
6. 新品。

## 八、评分规则
总分 100 分：
| 维度 | 分值 |
|---|---:|
| 问题归因质量 | 30 |
| 数据真实性与一致性 | 25 |
| 执行质量与闭环 | 20 |
| 经营结果表现 | 15 |
| 下月计划可落地性 | 10 |

## 九、最高分限制
| 情况 | 最高分 |
|---|---:|
| 总销售额和分渠道合计差异超过 5% 且无解释 | 60 |
| 店铺销售和达人/KOL/KOC销售混用且无说明 | 65 |
| 费用率无法复算且分母不清 | 70 |
| 关键数据前后冲突 | 70 |
| 未提供退款且无说明 | 75 |
| 没有上月动作闭环 | 80 |
| 只有结果没有归因 | 65 |
| 有明显模板错误、月份错误、字段未改 | 80 |
| 疑似重大数据造假或故意隐瞒 | 50，并建议重做 |

## 十、过渡期分流判断
1. 已标注当前主口径 + 有桥接说明 + 缺净成交额但有补齐计划：可判 A 或 B。
2. 未标注当前主口径 + 无桥接 + 多口径并存：判 C。
3. 主销售额无法判断 + 核心表格大量冲突：判 D。
